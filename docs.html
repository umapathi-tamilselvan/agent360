Section 01

Module Overview

What is AGENT 360?
AGENT 360 is the unified agent management module for ROSCA Digital. It gives branch managers, cluster admins, and org admins a complete, real-time view of every field agent — covering onboarding, performance scoring, commission tracking, introduction management, document KYC, and compliance locking. Field agents access their own 360 view via the mobile API portal.
👤

Agent Profile

Full KYC profile with personal info, addresses, identity numbers (PAN, Aadhaar, GST), bank details, and joining history.

⭐

Performance Score

Real-time 0–5 score based on introduction count vs. branch-tier targets and average enrollment quality scores.

💰

Commission Pipeline

Allocation → Disbursement → Refund lifecycle. Full audit trail per introduction with live commission balance.

🤝

Introduction Tracking

All subscriber enrollments introduced by this agent, with group, scheme, prize status, and per-enrollment commission breakdown.

🔐

Compliance & Lock

Admins can lock agents for compliance/fraud. Locked agents cannot receive commissions or add documents.

🔀

Agent Merge

Consolidate duplicate agent records — transfers introductions, documents, and relationships to the primary agent.

Section 07

Feature Set

👤

F01 — Agent Onboarding

Create and manage agent profiles with auto-generated codes.

  • Auto-generate code: {branch}-AGT-{n}
  • Full KYC form (identity, address, bank)
  • Assign managing employee
  • Set joining date
  • Provision portal access (optional)
⭐

F02 — Performance Scoring

Weighted 0–5 score calculated from introduction and enrollment metrics.

  • introductions_score (weight: 3) — 6-month rolling
  • enrollments_score (weight: 2) — avg quality
  • Tier targets: Urban 2 / Semi-Urban 1 / Rural 0.5
  • Auto-updated on enrollment changes
  • Batch recalc via UpdateAgentScoresJob
💰

F03 — Commission Engine

Full lifecycle: allocate → disburse → refund per introduction.

  • % defined in Group Policy per scheme
  • Allocation on enrollment activation
  • Disbursement to bank account
  • Refund on enrollment removal
  • Blocked if agent is locked
🤝

F04 — Introduction Tracking

See every subscriber enrolled by this agent across all groups.

  • Per-enrollment commission breakdown
  • Prize status tracking
  • Group + scheme info
  • Pagination, search, sort, filter
  • Export to report
📄

F05 — Document Management

Attach, view, and manage KYC documents for each agent.

  • PAN, Aadhaar, Ration card uploads
  • Document blocked if agent is locked
  • Documents transferred on merge
  • Polymorphic storage (shared infra)
🔐

F06 — Compliance Lock

Lock/unlock agents for compliance or fraud prevention.

  • Admin-only action (Cluster Admin+)
  • Lock reason stored with message
  • Blocks commission & documents
  • Audit logged with user + timestamp
  • UI shows lock badge and reason
🔀

F07 — Agent Merge

Consolidate duplicate agents into one canonical record.

  • Transfer all introductions
  • Transfer all documents
  • Transfer all relationships
  • Recalculate scores post-merge
  • Delete duplicate record
📱

F08 — Agent Portal (API)

Mobile / web app for agents to self-serve their 360 view.

  • View own introduction list
  • Commission balance and history
  • Account & KYC details
  • Filter by status, group, prize
  • Secure JWT + role auth

Section 12

Scoring Engine

Score Formula

// Final weighted score (0–5)
final_score = (
  enrollments_score × 2 +
  introductions_score × 3
) / 5

// Introductions Score (0–1)
intro_ratio = introductions_6mo / target
introductions_score = min(intro_ratio, 1.0)

// Branch Tier Targets (6 months)
Urban      → 2 introductions
Semi-Urban → 1 introduction
Rural      → 0.5 introductions

// Enrollments Score (0–1)
enrollments_score = avg(enrollment.score) / 5

Section 13

Commission Engine

📥

Allocation

When an enrollment becomes active, commission is calculated from agent_commission_percent in the Group Policy and credited to the agent's commission account.

💳

Disbursement

Commission transferred from agent account to bank account. Requires eligibility check (canDisburseIntroducerCommission()) to pass. Blocked if is_locked = true. Triggers transaction with purpose INTRODUCER_COMMISSION_DISBURSEMENT.

↩️

Refund

If an enrollment is removed, the allocated-but-undisbursed commission is refunded back using purpose INTRODUCER_COMMISSION_REFUND.

⚠️ Disbursement Eligibility — All conditions must pass before commission can be paid out.
Defined in Enrollment::canDisburseIntroducerCommission() and enforced by IntroducerCommissionDisbursementPolicy.

Disbursement Eligibility Conditions

#ConditionCode CheckFail Behaviour
1 2 completed auctions must exist in the group group.auctions.count() >= 2 Disbursement blocked — not yet eligible
2 Installments 1 & 2 fully paid (nil TBC) — the first 2 installments must have zero outstanding settlement amount Active enrollment:
installments.where('installment_no', '<=', 2)
.settlements.sum('to_be_settled_amount') == 0

Removed enrollment:
collected_amount >= auctions[1..2].sum('installment_amount') 		Disbursement blocked — commission carry-forwarded to next eligible month
3 Introducer not locked introducer.is_locked == false Exception thrown — commission stays allocated
4 Enrollment TBC = 0 (feature-flagged)
Only enforced when INTRODUCER_COMMISSION_DISBURSEMENT_WHEN_TBC=true		 enrollment.to_be_collected_amount == 0 Exception thrown — commission stays allocated until full collection
5 Enrollment not removed (policy gate) enrollment.status != 'removed' Policy denies disbursement

Eligibility Logic (from Enrollment.php)

public function canDisburseIntroducerCommission() {

  // Gate 1: group must have run at least 2 auctions
  $are_required_auctions_complete =
      $this->group->auctions->count() >= 2;

  // Gate 2: first 2 installments must be fully settled (nil TBC)
  if ($this->status == self::$STATUS_ACTIVE) {
    $ok = ($this->installments
              ->where('installment_no', '<=', 2)
              ->pluck('settlements')
              ->flatten()
              ->sum('to_be_settled_amount') == 0);
  } else {
    // Removed enrollment: compare collected vs required
    $installment_amount = $this->group->auctions
              ->where('installment_no', '<=', 2)
              ->sum('installment_amount');
    $ok = $this->collected_amount >= $installment_amount;
  }

  return $are_required_auctions_complete && $ok;
}
